# CONTEXT.md Format

## Structure

```md
# {Context Name}

{One or two sentence description of what this context is and why it exists.}

## Language

**Order**:
{A one or two sentence description of the term}
_Avoid_: Purchase, transaction

**Invoice**:
A request for payment sent to a customer after delivery.
_Avoid_: Bill, payment request
```

## Rules

- Be opinionated. Pick one canonical term when multiple words exist for the same concept.
- Keep definitions tight. One or two sentences max.
- Capture domain concepts, not implementation details.
- Add `_Avoid_` aliases when they help keep naming consistent.
