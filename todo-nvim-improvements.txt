Neovim Configuration Review - Improvements & Fixes
====================================================

Architecture Overview:
- Clean modular structure: init.lua -> core/ (options, keymaps, utils) + plugins/
- Modern Neovim 0.11+ native LSP API (vim.lsp.config/vim.lsp.enable)
- LSP well-organized: capabilities, diagnostics, on_attach, per-server configs

======================================================================
KEYBIND IDEAS
======================================================================

<leader>dB  - Conditional breakpoint (vim.fn.input("Condition: "))
<leader>du  - Toggle DAP UI manually (can't close UI without killing session)
<leader>uh  - Toggle inlay hints (global, more discoverable than <leader>th)
<leader>ud  - Toggle diagnostics (quick way to hide noise temporarily)
<leader>uf  - Toggle autoformat (useful in messy files)
<leader>xq  - Quickfix list (common workflow)
<C-d>/<C-u> + zz - Centered scrolling (less disorienting)
gx          - Open URL under cursor (ensure it works in your setup)
<leader>gp  - Git push (if using lazygit less frequently)
]q / [q     - Next/prev quickfix item
]t / [t     - Next/prev todo comment (if using todo-comments.nvim)
